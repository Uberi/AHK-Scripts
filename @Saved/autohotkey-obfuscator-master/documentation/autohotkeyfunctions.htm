Obfuscating Autohotkey System Function Calls Like dllcall()

Obfuscation of built in autohotkey functions like dllcall() can only be done if you use Dynamic Obfuscation. This is necessary because this type of object can only be obfuscated with the use of %'s, it can not be done with straight obfuscation.
 
Once you have set up basic Dynamic Obfuscation, there are just 2 commands that you will use to obfuscate built in functions.
 
;$OBFUSCATOR: $DEFSYSFUNCS: (list of functions, comma separated) { / {minobfstr}, {maxobfstr}, {obfrows}, {fragsperrow}, {minfragstr}, {maxfragstr} }
 
Use to specify a list of system functions to obfuscate, and use the optional parameters to set custom obfuscation encoding for the list. Parameters used this way will not change the obfuscation encoding defaults, only items in the list will be affected.
 
For an explanation of the meaning of each individual obfuscation parameter, refer to Understanding the Standard Parameter Set.
 
This obfuscator statement below has to be used somewhere in your source code before any built in function is used. It DUMPs assignment statements that will be used to initialize the variable fragments that will be used to obfuscate built in functions.
 
;$OBFUSCATOR: $SYSFUNCFRAGS_DUMPALL:
