Obfuscating Function Parameters in Autohotkey Scripts

This program has the ability to obfuscate your function parameters as used in your function headers and as used in your function bodies. The obfuscator will replace the parameters in the function header with obfuscation strings that contain no %'s because autohotkey does not allow them in this position. If Dynamic Obfuscation is being used then parameters found in the function body will be replaced with obfuscated strings that contain %'s, otherwise their replacements will have no %'s.

If Dynamic Obfuscation is Used, Use the DUMP Command for Parameters

If you use Dynamic Obfuscation, you must use the DUMP command for parameters. Since parameters are not allowed to be added to a secure class, there is one dump command for them that dumps the fragments for all the parameters in the program and it doesn't allow any parameters:
 
;$OBFUSCATOR: $PARAMFRAGS_DUMPALL:
 
You must use the DUMP statements for common object name fragments before you use the statement above because common object fragments will be inserted into the DUMPed code.

Change Parameter Obfuscation Defaults

In order to change the obfuscation encoding defaults for parameters, you use the change defaults command for parameters.
 
;$OBFUSCATOR: $PARAMS_CHANGE_DEFAULTS: {minobfstr}, {maxobfstr}, {obfrows}, {fragsperrow}, {minfragstr}, {maxfragstr}
 
For an explanation of the meaning of each individual obfuscation parameter, refer to Understanding the Standard Parameter Set.
 
You can also use the restore defaults command to restore what they were before the change. Only one restore step back is allowed.
 
;$OBFUSCATOR: $PARAMS_RESTORE_DEFAULTS:
