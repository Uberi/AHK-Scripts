Obfuscating Global Variables in Your Autohotkey Script

This program has the ability to obfuscate your global variables anywhere they are used. If Dynamic Obfuscation is being used then all the usages of global variables found will be replaced with obfuscated strings that contain %'s, otherwise their replacements will have no %'s.

Global Variables Are NOT Automatically Obfuscated

An obfuscation is not automatically created for global variables. In order to have them obfuscated, you specify their names in the define global variables command comment:
 
;$OBFUSCATOR: $DEFGLOBVARS: (list of global variable names) { / {minobfstr}, {maxobfstr}, {obfrows}, {fragsperrow}, {minfragstr}, {maxfragstr} }
 
It does not matter where you put these statements in the source code. They do not need to be put before your usage of the global variables or in the section that the variables are used in.
 
If you want, you can change the obfuscation encoding parameters for the variables by using the second optional part of the statement. If the optional parameters part of the statement is used, it is preceeded with a '/'. If used, they will change the defaults for only the variables listed in the statement. For more information on usage of the individual parameters, refer to the obfuscation command comments section

If Dynamic Obfuscation is Used, Use the DUMP Command for Global Variables

If you use Dynamic Obfuscation, you must use the DUMP command for global variables. Since global variables can not be added to a secure class, there is one dump command for them that dumps all the fragments for all the global variables in the program and it doesn't allow any parameters:
 
;$OBFUSCATOR: $GLOBVARFRAGS_DUMPALL:
 
You must use the DUMP statements for common object name fragments before you use the statement above because common object fragments will be inserted into the DUMPed code.

Change Global Variable Obfuscation Defaults

In order to change the obfuscation encoding defaults for global variables, you use the change defaults command for them.
 
;$OBFUSCATOR: $GLOBVARS_CHANGE_DEFAULTS: {minobfstr}, {maxobfstr}, {obfrows}, {fragsperrow}, {minfragstr}, {maxfragstr}
 
For an explanation of the meaning of each individual obfuscation parameter, refer to Understanding the Standard Parameter Set.
 
You can also use the restore defaults command to restore what they were before the change. Only one restore step back is allowed.
 
;$OBFUSCATOR: $GLOBVARS_RESTORE_DEFAULTS:
