Obfuscating Local and Static Variables in Autohotkey Scripts

This program has the ability to obfuscate your local and static variables as used in your functions. The obfuscator will replace the first instance of a local or static variable found in the function with obfuscation strings that contain no %'s. This is because it is assumed they are in a LOCAL or STATIC statement and autohotkey does not allow %'s in those statements. If Dynamic Obfuscation is being used then the rest of the usage of that local or static variable found in the function body will be replaced with obfuscated strings that contain %'s, otherwise their replacements will have no %'s.

Local and Static Variables ARE NOT Automatically Obfuscated

An obfuscation is not automatically created for local and static variables. In order to have them obfuscated, you specify their names in the define local or static variables command comment:
 
;$OBFUSCATOR: $DEFLOSVARS: (list of local and static variable names) { / {minobfstr}, {maxobfstr}, {obfrows}, {fragsperrow}, {minfragstr}, {maxfragstr} }
 
The local and static variables defined in that statement will be associated with the most recently found function so these statements should be placed in the relevant function.
 
If you want, you can change the obfuscation encoding parameters for the variables by using the second optional part of the statement. If the optional parameters part of the statement is used, it is preceeded with a '/'. If used, they will change the defaults for only the variables listed in the statement. For more information on usage of the individual parameters, refer to understanding the standard parameter set.

If Dynamic Obfuscation is Used, Use the DUMP Command for Local and Static Variables

If you use Dynamic Obfuscation, you must use the DUMP command for local and static variables. Since local and static variables can not be added to a secure class, there is one dump command for them that dumps all the fragments for all the local and static variables in the program and it doesn't allow any parameters:
 
;$OBFUSCATOR: $LOSVARFRAGS_DUMPALL:
 
You must use the DUMP statements for common object name fragments before you use the statement above because common object fragments will be inserted into the DUMPed code.

Change Local and Static Variable Obfuscation Defaults

In order to change the obfuscation encoding defaults for local or static variables, you use the change defaults command for them:
 
;$OBFUSCATOR: $LOSVARS_CHANGE_DEFAULTS: {minobfstr}, {maxobfstr}, {obfrows}, {fragsperrow}, {minfragstr}, {maxfragstr}
 
For an explanation of the meaning of each individual obfuscation parameter, refer to Understanding the Standard Parameter Set.
 
You can also use the restore defaults command to restore what they were before the change. Only one restore step back is allowed.
 
;$OBFUSCATOR: $LOSVARS_RESTORE_DEFAULTS:
